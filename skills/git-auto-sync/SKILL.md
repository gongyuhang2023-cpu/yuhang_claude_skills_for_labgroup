# Git Auto Sync - 一键 Git 同步技能

## 作用

自动执行完整的项目同步流程：
1. **更新项目目录**（如存在 `项目目录.md`）
2. **Git 同步**：`git add .` → `git commit` → `git push`

## 使用方法

### 方式一：运行脚本

```bash
# 使用自动生成的提交信息
python ~/.claude/skills/git-auto-sync/scripts/sync.py

# 指定提交信息
python ~/.claude/skills/git-auto-sync/scripts/sync.py "修复了登录bug"
```

### 方式二：对话触发

在对话中说 "git 保存并推送"、"同步项目到远程"、"git-auto-sync" 或 "提交代码"。

## 执行流程（顺序工作流模式）

Claude 执行此技能时，**必须按以下顺序操作**：

### 步骤 0：科研实验项目一致性检测

> **适用范围**：仅限**科研实验项目**（湿实验、制剂制备、细胞/动物实验等）
> **不适用**：数据分析项目、生物信息学项目、纯代码项目

#### 0.1 判断项目类型

**科研实验项目的特征**（需同时满足）：
| 特征 | 检测方式 |
|------|----------|
| CLAUDE.md 包含「一致性维护」章节 | `grep "一致性维护" ./CLAUDE.md` |
| 存在 `Experiments/` 文件夹 | 目录结构检查 |
| 存在 `protocol/` 或 `Protocols/` | 目录结构检查 |

**数据分析项目的特征**（排除条件）：
| 特征 | 说明 |
|------|------|
| 存在 `_quarto.yml` | Quarto 数据分析项目 |
| 存在 `analyses/` 但无 `Experiments/` | 纯分析项目 |
| CLAUDE.md 包含「图表规范」但无「一致性维护」 | bioinfo 项目 |

**如果不是科研实验项目，跳过此步骤，直接执行步骤 1。**

#### 0.2 扫描本次变更

```bash
git diff --name-only HEAD
git diff --name-only --cached
```

#### 0.3 一致性提醒规则

| 变更文件类型 | 可能需要同步的文件 |
|--------------|-------------------|
| `**/protocol/*.md` 或 `Protocols/*.md` | 实验计划.md, CLAUDE.md（如涉及核心结构） |
| `实验计划.md` | CLAUDE.md, 各实验 protocol |
| `CLAUDE.md`（核心结构部分） | 实验计划.md, protocol/ |
| `Experiments/*/*.md`（实验记录） | 项目目录.md（由脚本自动更新） |

#### 0.4 输出提醒

```
🔬 科研实验项目一致性检测

检测到以下关键文件变更：
- Experiments/Exp01_xxx/protocol/01_脂质体制备SOP.md (修改)

⚠️ 请确认是否需要同步更新：
- [ ] 实验计划.md - 检查实验方案描述
- [ ] CLAUDE.md - 如涉及核心结构定义

确认无需同步 / 已完成同步后，继续执行 Git 提交。
```

**注意**：此步骤仅为**提醒**，不阻止提交流程。

---

### 步骤 1：更新项目目录（可选）

```bash
python ~/.claude/skills/git-auto-sync/scripts/update_directory.py
```

**容错规则**：
- 如果 `项目目录.md` 不存在，脚本会跳过
- 如果更新失败，脚本仍返回成功（退出码 0）
- **无论此步骤结果如何，必须继续执行后续 Git 操作**

### 步骤 2：执行 Git 同步

```bash
python ~/.claude/skills/git-auto-sync/scripts/sync.py "提交信息"
```

或手动执行：
```bash
git add .
git commit -m "提交信息"
git push
```

## 脚本说明

### sync.py - Git 同步主脚本

执行 `git add .` → `git commit` → `git push` 流程。

**功能**：
- 检查是否为 Git 仓库
- 检查是否有变更
- 添加所有变更到暂存区
- 提交（用户指定或自动生成提交信息）
- 推送到远程

**自动生成的提交信息格式**：
```
Claude 自动同步于 2025-01-12 15:30:45
```

### update_directory.py - 项目目录更新脚本

自动更新 `项目目录.md` 文件的时间戳和更新日志。

**功能**：
- 扫描 git 变更信息
- 更新文档顶部时间戳
- 在更新日志表格中添加新条目
- 记录本次变更摘要（新增/修改/删除文件数）

**容错设计**：
- 如果 `项目目录.md` 不存在，静默跳过
- 任何错误都不会阻止后续 git 操作
- 始终返回退出码 0

---

## 错误处理

| 错误类型 | 处理方式 |
|---------|---------|
| 目录更新失败 | 打印警告，继续 Git 操作 |
| 非 Git 仓库 | 提示用户先初始化 Git |
| 无变更 | 提示没有需要提交的内容 |
| 推送冲突 | 提示用户手动拉取并解决冲突 |
| 网络错误 | 提示检查网络连接 |

## 项目目录功能

### 目的与用途

`项目目录.md` 是 **Claude 的项目导航索引**（Agentic Navigation Index），用于：
- **快速定位文件**：通过完整路径直接找到目标文件
- **了解项目架构**：无需全量扫描即可掌握项目结构
- **减少 Token 消耗**：优先查阅索引而非执行递归搜索
- **追踪项目变动**：了解最近的更新上下文

**重要准则**：Claude 在执行任何复杂的文件操作或搜索之前，**必须先读取 `项目目录.md`** 以缩小搜索范围。

### 项目目录.md 内容规范

为实现高效导航，目录文件应包含以下内容（低自由度格式，便于解析）：

```markdown
# 项目目录

> 自动生成于 2026-01-21

## 项目架构 (Architecture Overview)

| 文件夹 | 路径 | 职能 |
|--------|------|------|
| 实验记录 | `./Experiments/` | 存放所有实验的 md 记录文件 |
| 数据文件 | `./Experiments/Data/` | 原始数据、处理脚本、结果图 |
| 实验方案 | `./Protocols/` | SOP 和参考手册 (PDF + md) |

## 关键文件锚点 (Key File Anchors)

| 文件 | 路径 | 用途 | 标签 |
|------|------|------|------|
| 噬菌体分离 | `./Experiments/01_噬菌体分离.md` | R1-R3, W1-W2 分离记录 | #实验 #已完成 |
| 杀菌曲线数据 | `./Experiments/Data/08_杀菌曲线/` | CSV原始数据 + R脚本 | #数据 #可视化 |

## 最近变动 (Recent Context)

| 日期 | 变动内容 |
|------|----------|
| 2026-01-21 | 新增杀菌曲线 R 菌数据和可视化 |

## 更新日志

| 日期 | 更新内容 |
|------|----------|
```

### 内容维护要求

Claude 维护此文件时应确保：
1. **完整相对路径** - 使用 `./` 开头的相对路径
2. **单句功能描述** - 每个条目简明扼要
3. **检索标签** - 使用 `#标签` 便于关键词搜索
4. **及时更新** - 新增/删除文件时同步更新

### 自动更新机制

脚本会自动：
1. 更新 "自动生成于" 时间戳
2. 在更新日志表格中添加变更摘要

**注意**：脚本只更新时间戳和日志，目录内容由 Claude 手动维护以确保准确性。

### 在 CLAUDE.md 中添加导航准则

建议在项目的 `CLAUDE.md` 中添加以下内容：

```markdown
## 项目导航准则

本项目使用 `项目目录.md` 作为核心索引。

**重要**：在寻找文件时，请优先查阅此文件而非执行全局递归搜索，以节省 Token 并提高准确性。

`git-auto-sync` 技能会在每次同步时自动更新该文件的时间戳。
```

### 不需要项目目录

如果项目根目录没有 `项目目录.md`，脚本会自动跳过目录更新步骤，直接执行 Git 同步。

---

## 注意事项

- 脚本会添加所有变更（包括新文件和修改）
- 推送前请确保已设置远程仓库
- 如遇冲突，需手动执行 `git pull` 解决后再推送
- 项目目录更新是可选功能，不影响核心 Git 同步
